# nodemcu_ip_weather_clock

Скетч для NodeMCU 1.0 ESP-12E / ESP8266:

- подключается к WiFi;
- синхронизирует время через NTP;
- опрашивает NTP не чаще одного раза в 30 секунд;
- определяет город и координаты по внешнему IP;
- получает погоду OpenWeatherMap по найденным координатам раз в минуту;
- поднимает HTTP-сервер на порту `80`;
- показывает дату, время и погодные данные на веб-странице.

DS18B20 в этой версии не используется.

## Распиновка NodeMCU

Для этой версии дополнительные датчики не подключаются. Достаточно питания NodeMCU и WiFi.

Источник: [Random Nerd Tutorials: ESP8266 Pinout Reference](https://randomnerdtutorials.com/esp8266-pinout-reference-gpios/).

## Настройки

В начале файла `nodemcu_ip_weather_clock.ino` задайте:

- `WIFI_SSID`
- `WIFI_PASSWORD`
- `OPENWEATHER_API_KEY`
- `NTP_SERVER`
- `TIME_OFFSET_SECONDS`

По умолчанию указано смещение `UTC+3`:

```cpp
const long TIME_OFFSET_SECONDS = 3L * 60L * 60L;
```

Координаты по умолчанию используются как fallback, если геолокация по IP еще не получена:

```cpp
const float DEFAULT_WEATHER_LATITUDE = 55.567586;
const float DEFAULT_WEATHER_LONGITUDE = 38.225004;
const char DEFAULT_WEATHER_CITY[] = "Ramenskoye";
```

Это Раменское, Московская область.

Геолокация по IP обновляется раз в 6 часов:

```cpp
const unsigned long GEOLOCATION_UPDATE_INTERVAL = 6UL * 60UL * 60UL * 1000UL;
```

## Веб-страница

После загрузки откройте Serial Monitor на скорости `115200` и найдите IP-адрес NodeMCU.

Откройте в браузере:

```text
http://<ip-address>/
```

Дата выводится в `h3`, время в `h2`:

```html
<h3>9 мая 2026, суббота</h3>
<h2>14:25</h2>
```

Автообновления страницы нет.

Дополнительно выводятся:

- город, для которого берется погода;
- координаты погоды;
- город, страна и внешний IP, определенные по IP;
- статус последнего обновления геолокации.

## Геолокация по IP

Запрос выполняется к `ip-api.com`:

```text
http://ip-api.com/json/?fields=status,message,country,city,lat,lon,query
```

После успешного ответа найденные `lat` и `lon` используются для запроса погоды OpenWeatherMap. Если сервис недоступен, погода берется по координатам fallback.

## Погода

Запрос выполняется к OpenWeatherMap:

```text
http://api.openweathermap.org/data/2.5/weather
```

Интервал запроса:

```cpp
const unsigned long WEATHER_UPDATE_INTERVAL = 5UL * 60UL * 1000UL;
```

Если OpenWeatherMap не ответил, повторная попытка выполняется быстрее:

```cpp
const unsigned long WEATHER_RETRY_INTERVAL = 30UL * 1000UL;
```

Показываются:

- статус последнего обновления OpenWeatherMap: `OK`, `HTTP <код>`, ошибка JSON/HTTP init или ошибка WiFi;
- температура;
- ощущаемая температура;
- `main` и `description`;
- влажность;
- давление в мм рт. ст.;
- сглаженное давление после EMA-фильтра;
- скорость изменения давления в мм рт. ст./ч;
- простой прогноз по давлению с 5 уровнями;
- ASCII-график сглаженного давления;
- скорость ветра;
- направление ветра в градусах;
- ASCII-роза ветров по секторам 10 градусов.

Давление OpenWeatherMap приходит в гПа и переводится в мм рт. ст. по коэффициенту `0.750062`.

Если очередное обновление OpenWeatherMap завершилось ошибкой, последние успешно полученные значения и накопленная история остаются на странице. Меняется только строка статуса последнего обновления.

Давление фильтруется экспоненциальным скользящим средним: `новое = текущее * k + предыдущее * (1 - k)`. Коэффициент задается в `PRESSURE_SMOOTHING_K`, по умолчанию `0.3`. Новая сглаженная точка добавляется в историю после каждого успешного обновления OpenWeatherMap. Скорость изменения считается как разница между текущим сглаженным давлением и сглаженным давлением примерно 30 минут назад, пересчитанная в мм рт. ст./ч.

История сглаженного давления и история направлений ветра сохраняются в EEPROM. При рестарте буферы загружаются обратно. Если EEPROM пустой или контрольная сумма не совпала, при первом успешном получении давления история заполняется текущим значением, чтобы график и скорость не стартовали с пустой истории.

История ветра скользящая за последние 24 часа. При интервале обновления погоды 5 минут хранится до 288 последних направлений. Роза ветров пересчитывает статистику по секторам 10 градусов: `0`, `10`, `20`, ..., `350`.

Уровни прогноза:

- `▼▼` быстро падает;
- `▼` падает;
- `■` стабильно;
- `▲` растет;
- `▲▲` быстро растет.

Пороги скорости:

```cpp
const float PRESSURE_FAST_SPEED_THRESHOLD = 1.5;
const float PRESSURE_SLOW_SPEED_THRESHOLD = 0.2;
```

ASCII-график строится по истории сглаженного давления. Старые значения находятся слева, новые справа: `-3h`, `-2h`, `-1h`, `0`. Давление идет по вертикали, шаг по умолчанию `0.1` мм рт. ст.; если перепад слишком большой, шаг автоматически укрупняется, чтобы график помещался в 30 строк.

```text
747.2 |     **
747.1 |   **  
747.0 | **    
746.9 |*      
      +------------------------------------
       -3h         -2h         -1h       0
```

ASCII-роза ветров строится по скользящей истории направлений за последние 24 часа. Центр отмечен `+`, стороны света `N/E/S/W`, лучи `*`. Длина луча пропорциональна количеству попаданий в сектор:

```text
          N          
       *** ***       
     **   +   **     
W   **         **   E
     **       **     
       *******       
          S          
samples: 120, max sector: 18
```

## Библиотеки Arduino IDE

Установите поддержку ESP8266 Boards и выберите плату `NodeMCU 1.0 (ESP-12E Module)`.

Используемые библиотеки:

- ESP8266WiFi
- ESP8266WebServer
- WiFiUdp
- NTPClient
- ArduinoJson
- EEPROM

`NTPClient` установите через Library Manager.
`ArduinoJson` установите через Library Manager.
