# wemos_ntp_web_clock

Скетч для Wemos D1 mini / ESP8266:

- подключается к WiFi;
- синхронизирует время через NTP;
- опрашивает NTP не чаще одного раза в 30 секунд;
- получает погоду OpenWeatherMap для Раменского раз в минуту;
- читает DS18B20 на GPIO0 / D3 раз в минуту и при запуске;
- поднимает HTTP-сервер на порту `80`;
- показывает дату, время и погодные данные на веб-странице.

## Настройки

В начале файла `wemos_ntp_web_clock.ino` задайте:

- `WIFI_SSID`
- `WIFI_PASSWORD`
- `OPENWEATHER_API_KEY`
- `NTP_SERVER`
- `TIME_OFFSET_SECONDS`

По умолчанию указано смещение `UTC+3`:

```cpp
const long TIME_OFFSET_SECONDS = 3L * 60L * 60L;
```

Координаты по умолчанию:

```cpp
const float WEATHER_LATITUDE = 55.567586;
const float WEATHER_LONGITUDE = 38.225004;
```

Это Раменское, Московская область.

## Веб-страница

После загрузки откройте Serial Monitor на скорости `115200` и найдите IP-адрес Wemos.

Откройте в браузере:

```text
http://<ip-address>/
```

Дата выводится в `h3`, время в `h2`:

```html
<h3>3 May 2026</h3>
<h2>14:25:09</h2>
```

Страница обновляется раз в 1 секунду через meta refresh.

## Погода

Запрос выполняется к OpenWeatherMap:

```text
http://api.openweathermap.org/data/2.5/weather
```

Интервал запроса:

```cpp
const unsigned long WEATHER_UPDATE_INTERVAL = 60UL * 1000UL;
```

Показываются:

- статус последнего обновления OpenWeatherMap: `OK`, `HTTP <код>`, ошибка JSON/HTTP init или ошибка WiFi;
- температура;
- ощущаемая температура;
- `main` и `description`;
- влажность;
- давление в мм рт. ст.;
- среднее давление за текущее 5-минутное окно;
- скорость изменения давления в мм рт. ст./ч;
- простой прогноз по давлению с 5 уровнями;
- ASCII-график завершенных 5-минутных окон давления;
- температура DS18B20 и среднее за 5 минут;
- скорость ветра;
- направление ветра в градусах.
- ASCII-роза ветров по секторам 10 градусов.

Давление OpenWeatherMap приходит в гПа и переводится в мм рт. ст. по коэффициенту `0.750062`.

Если очередное обновление OpenWeatherMap завершилось ошибкой, последние успешно полученные значения остаются на странице. Меняется только строка статуса последнего обновления.

Давление усредняется в окнах по 5 минут. Завершенные 5-минутные окна хранятся за последние 3 часа. Скорость изменения считается как разница между текущим средним давлением и давлением примерно 30 минут назад, пересчитанная в мм рт. ст./ч.

История 5-минутных окон давления и история направлений ветра сохраняются в EEPROM. При рестарте буферы загружаются обратно. Если EEPROM пустой или контрольная сумма не совпала, при первом успешном получении давления все 36 окон заполняются текущим значением, чтобы график и скорость не стартовали с пустой истории.

История ветра скользящая за последние 24 часа. При интервале обновления погоды 1 минута хранится до 1440 последних направлений. Роза ветров пересчитывает статистику по секторам 10 градусов: `0`, `10`, `20`, ..., `350`.

Уровни прогноза:

- `▼▼` быстро падает;
- `▼` падает;
- `■` стабильно;
- `▲` растет;
- `▲▲` быстро растет.

Пороги скорости:

```cpp
const float PRESSURE_FAST_SPEED_THRESHOLD = 1.5;
const float PRESSURE_SLOW_SPEED_THRESHOLD = 0.2;
```

ASCII-график строится по завершенным 5-минутным окнам. Старые значения находятся слева, новые справа. Давление идет по вертикали, шаг по умолчанию `0.1` мм рт. ст.; если перепад слишком большой, шаг автоматически укрупняется, чтобы график помещался в 30 строк.

```text
747.2 |     **
747.1 |   **  
747.0 | **    
746.9 |*      
      +-------
       old new
```

ASCII-роза ветров строится по скользящей истории направлений за последние 24 часа. Центр отмечен `+`, стороны света `N/E/S/W`, лучи `*`. Длина луча пропорциональна количеству попаданий в сектор:

```text
          N          
       *** ***       
     **   +   **     
W   **         **   E
     **       **     
       *******       
          S          
samples: 120, max sector: 18
```

## DS18B20

Датчик подключен к GPIO0, это пин `D3` на Wemos D1 mini:

```cpp
const byte DS18B20_DATA_PIN = 0;
```

Чтение выполняется при запуске и дальше раз в минуту:

```cpp
const unsigned long TEMPERATURE_READ_INTERVAL = 60UL * 1000UL;
```

Средняя температура считается за окно 5 минут:

```cpp
const unsigned long TEMPERATURE_AVERAGE_WINDOW = 5UL * 60UL * 1000UL;
```

В коде включена внутренняя подтяжка ESP8266:

```cpp
pinMode(DS18B20_DATA_PIN, INPUT_PULLUP);
```

Это может работать без внешнего резистора на коротком проводе, но для стабильной шины 1-Wire обычно нужен внешний резистор 4.7 кОм между DATA и 3.3V.
