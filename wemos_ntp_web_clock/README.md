# wemos_ntp_web_clock

Скетч для Wemos D1 mini / ESP8266:

- подключается к WiFi;
- синхронизирует время через NTP;
- опрашивает NTP не чаще одного раза в 30 секунд;
- определяет геолокацию по внешнему IP и использует ее для OpenWeatherMap;
- получает погоду OpenWeatherMap раз в 5 минут, с повтором через 30 секунд после ошибки;
- читает DS18B20 на GPIO0 / D3 раз в минуту и при запуске;
- поднимает HTTP-сервер на порту `80`;
- показывает дату, время, статусы геолокации/OWM, последние погодные данные и DS18B20 на веб-странице.

## Настройки

В начале файла `wemos_ntp_web_clock.ino` задайте:

- `WIFI_SSID`
- `WIFI_PASSWORD`
- `OPENWEATHER_API_KEY`
- `NTP_SERVER`
- `TIME_OFFSET_SECONDS`

По умолчанию указано смещение `UTC+3`:

```cpp
const long TIME_OFFSET_SECONDS = 3L * 60L * 60L;
```

Координаты по умолчанию, если IP-геолокация еще не получена:

```cpp
const float DEFAULT_WEATHER_LATITUDE = 55.567586;
const float DEFAULT_WEATHER_LONGITUDE = 38.225004;
```

Это Раменское, Московская область.

## Веб-страница

После загрузки откройте Serial Monitor на скорости `115200` и найдите IP-адрес Wemos.

Откройте в браузере:

```text
http://<ip-address>/
```

Дата выводится в `h3`, время в `h2`:

```html
<h3>3 May 2026</h3>
<h2>14:25</h2>
```

Автообновление страницы через meta refresh отключено; время выводится без секунд.

## Погода

Запрос выполняется к OpenWeatherMap:

```text
http://api.openweathermap.org/data/2.5/weather
```

Интервал запроса:

```cpp
const unsigned long WEATHER_UPDATE_INTERVAL = 5UL * 60UL * 1000UL;
const unsigned long WEATHER_RETRY_INTERVAL = 30UL * 1000UL;
```

IP-геолокация обновляется раз в 6 часов:

```cpp
const unsigned long GEOLOCATION_UPDATE_INTERVAL = 6UL * 60UL * 60UL * 1000UL;
```

Показываются:

- город и координаты погоды, полученные через IP-геолокацию или fallback-координаты;
- статус последнего обновления геолокации: `OK`, `HTTP <код>`, ошибка JSON/HTTP init или ошибка WiFi;
- статус последнего обновления OpenWeatherMap: `OK`, `HTTP <код>`, ошибка JSON/HTTP init или ошибка WiFi;
- температура;
- ощущаемая температура;
- `main` и `description`;
- влажность;
- давление в мм рт. ст.;
- сглаженное давление по EMA-фильтру;
- скорость изменения давления в мм рт. ст./ч;
- простой прогноз по давлению с 5 уровнями;
- ASCII-график сглаженного давления за последние 3 часа;
- температура DS18B20 и сглаженное значение по EMA-фильтру;
- скорость ветра;
- направление ветра в градусах.
- ASCII-роза ветров по секторам 10 градусов.

Давление OpenWeatherMap приходит в гПа и переводится в мм рт. ст. по коэффициенту `0.750062`.

Если очередное обновление геолокации или OpenWeatherMap завершилось ошибкой, последние успешно полученные значения остаются на странице. Меняется только строка статуса последнего обновления.

Давление сглаживается EMA-фильтром с коэффициентом `PRESSURE_SMOOTHING_K = 0.3`. Сглаженные сэмплы хранятся за последние 3 часа. Скорость изменения считается как разница между текущим сглаженным давлением и давлением примерно 30 минут назад, пересчитанная в мм рт. ст./ч.

История сглаженного давления и история направлений ветра сохраняются в EEPROM. При рестарте буферы загружаются обратно. Если EEPROM пустой или контрольная сумма не совпала, при первом успешном получении давления все 36 сэмплов заполняются текущим значением, чтобы график и скорость не стартовали с пустой истории.

История ветра скользящая за последние 24 часа. При интервале обновления погоды 5 минут хранится до 288 последних направлений. Роза ветров пересчитывает статистику по секторам 10 градусов: `0`, `10`, `20`, ..., `350`.

Уровни прогноза:

- `▼▼` быстро падает;
- `▼` падает;
- `■` стабильно;
- `▲` растет;
- `▲▲` быстро растет.

Пороги скорости:

```cpp
const float PRESSURE_FAST_SPEED_THRESHOLD = 1.5;
const float PRESSURE_SLOW_SPEED_THRESHOLD = 0.2;
```

ASCII-график строится по сглаженным сэмплам. Старые значения находятся слева, новые справа. Давление идет по вертикали, шаг по умолчанию `0.1` мм рт. ст.; если перепад слишком большой, шаг автоматически укрупняется, чтобы график помещался в 30 строк.

```text
747.2 |              **
747.1 |          ****
747.0 |     *****
746.9 |*****
      +----------------
       -3h  -2h  -1h 0
```

ASCII-роза ветров строится по скользящей истории направлений за последние 24 часа. Центр отмечен `+`, стороны света `N/E/S/W`, лучи `*`. Длина луча пропорциональна количеству попаданий в сектор:

```text
          N
       *** ***
     **   +   **
W   **         **   E
     **       **
       *******
          S
samples: 120, max sector: 18
```

## DS18B20

Датчик подключен к GPIO0, это пин `D3` на Wemos D1 mini:

```cpp
const byte DS18B20_DATA_PIN = 0;
```

Чтение выполняется при запуске и дальше раз в минуту:

```cpp
const unsigned long TEMPERATURE_READ_INTERVAL = 60UL * 1000UL;
```

Температура DS18B20 сглаживается EMA-фильтром:

```cpp
const float TEMPERATURE_SMOOTHING_K = 0.3;
```

В коде включена внутренняя подтяжка ESP8266. Если очередное чтение завершилось ошибкой, веб-страница показывает ошибку рядом с последним успешным значением, если оно уже было получено; сглаженное EMA-значение при ошибке не пересчитывается.

Внутренняя подтяжка:

```cpp
pinMode(DS18B20_DATA_PIN, INPUT_PULLUP);
```

Это может работать без внешнего резистора на коротком проводе, но для стабильной шины 1-Wire обычно нужен внешний резистор 4.7 кОм между DATA и 3.3V.
